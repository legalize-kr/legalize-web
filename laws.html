법령 데이터

대한민국 법령을 법령명 기준의 Markdown 파일로 정리하고, 공포일자와 시행일자를 메타데이터로 보존합니다. 개정 이력은 Git 커밋으로 추적됩니다.

개요

법령 데이터는 국가법령정보센터 OpenAPI의 원문을 Markdown으로 변환한 결과입니다. 법률, 시행령, 시행규칙, 대통령령, 대법원규칙 등 법령 종류를 구분하고, 같은 법 체계에 속하는 문서를 하나의 디렉토리에서 함께 관리합니다.

파일 본문은 사람이 읽기 쉬운 Markdown이고, 제목·법령구분·소관부처· 공포일자·시행일자 같은 필드는 YAML frontmatter에 들어갑니다. 특정 날짜 기준의 법령을 조회하거나 개정 전후를 비교할 때 이 구조를 그대로 사용할 수 있습니다.

데이터 구조

법령은 법령명 디렉토리 아래에 법령 종류별 파일로 배치됩니다. 관련 법률, 시행령, 시행규칙이 존재하면 같은 디렉토리에서 함께 찾을 수 있습니다.

법률
kr/{법률명}/법률.md
시행령
kr/{법률명}/시행령.md
시행규칙
kr/{법률명}/시행규칙.md
독립 대통령령
kr/{대통령령명}/대통령령.md

# 민법

## 제1편 총칙

Git 활용 방법

법령 저장소를 clone하면 일반 텍스트 저장소처럼 git log, git diff, grep 명령어를 사용할 수 있습니다.

법령 Git 활용 
git log -- kr/근로기준법/
git diff HEAD~1 -- kr/민법/법률.md
grep -r "개인정보" kr/
git log --before="2025-01-01" -1 -- kr/민법/법률.md
git log --grep="^법률:"

커밋 메시지

법령 커밋은 공포일자를 Git author/committer date로 사용하며, 원문 URL, 공포번호, 소관부처, 법령MST 같은 추적 정보를 포함합니다.

예시: 법률 개정 커밋 
법률: 민법 (일부개정)

법령 전문: https://www.law.go.kr/법령/민법
제개정문: https://www.law.go.kr/법령/제개정문/민법/(12345,20260317)
신구법비교: https://www.law.go.kr/법령/신구법비교/민법

공포일자: 2026-03-17 | 공포번호: 12345
소관부처: 법무부
법령MST: 284415

CLI 사용법

legalize-cli는 GitHub REST API로 법령 파일과 이력을 조회합니다. 자동화나 LLM 연동에는 --json 옵션을 사용합니다. 상세한 설치와 옵션은 cli-tools 저장소를 참고하세요.

법령 CLI 
pip install legalize-cli
legalize laws list --category 법률 --json
legalize laws get 민법 --date 2015-06-01
legalize laws article 민법 제839조의2 --date 2015-06-01 --json
legalize laws diff 민법 민법 --date-a 2015-01-01 --date-b 2024-01-01 --mode article

MCP 사용법

MCP 서버에서는 법령 목록, 전문, 조문, 통합 검색 도구를 제공합니다. Claude Desktop이나 Cursor 설정에서 legalize-mcp를 stdio 서버로 등록합니다. MCP 서버 설치와 클라이언트 설정은 cli-tools 저장소를 참고하세요.

목록

laws_list로 법령 종류와 페이지를 지정해 조회합니다.

전문

laws_get으로 법령명과 기준일자를 지정합니다.

조문

laws_article로 특정 조문만 추출합니다.

활용 사례

개정 전후 비교

동일 법령의 두 날짜를 지정해 조문 단위 변경 사항을 확인합니다.

법령 체계 검색

특정 표현이나 제도명을 전체 법령에서 검색해 관련 법령을 찾습니다.

서비스 데이터셋

YAML 메타데이터와 Markdown 본문을 분리해 검색 색인이나 RAG에 사용합니다.
