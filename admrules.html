행정규칙 데이터

중앙행정기관과 소속기관이 발령하는 고시, 훈령, 예규, 공고, 지침을 기관 계층과 행정규칙 종류별 Markdown 파일로 제공합니다.

개요

행정규칙 데이터는 중앙행정기관의 집행 기준과 내부 기준을 Git으로 관리하기 위한 데이터셋입니다. 법령이 위임한 세부 기준, 기관별 업무 처리 기준, 고시·훈령·예규 형태의 운영 규정을 함께 살펴볼 수 있습니다.

발령일자를 기준 날짜로 사용하고, 실제 발령 주체의 기관 계층을 경로에 보존합니다. 기관명으로 좁히거나 행정규칙 종류로 좁히는 조회가 모두 가능합니다.

데이터 구조

경로는 상위 기관, 본부 또는 소속기관, 행정규칙 종류, 행정규칙명 순서입니다. 기관 개편 전후의 명칭은 원문과 메타데이터를 기준으로 보존합니다.

1단계: 상위 기관

고용노동부, 국무총리, 과학기술정보통신부처럼 상위 기관명을 최상위 디렉토리로 둡니다.

2단계: 발령 주체

본부 발령 문서는 _본부에 두고, 소속기관 발령 문서는 고용노동부고객상담센터, 국무조정실처럼 실제 발령 기관명으로 구분합니다.

3단계: 행정규칙 종류

고시, 훈령, 예규, 공고, 지침, 국무총리훈령, 대통령훈령 등을 별도 디렉토리로 분리합니다.

4단계: 문서 본문

행정규칙명 디렉토리 안의 본문.md가 실제 Markdown 본문입니다. 기관명과 종류가 같아도 문서명별로 분리됩니다.

고시
행정안전부/_본부/고시/{명}/본문.md
훈령
국무총리/_본부/훈령/{명}/본문.md
예규
고용노동부/_본부/예규/{명}/본문.md
공고·지침
{기관}/_본부/{종류}/{명}/본문.md
소속기관 훈령
고용노동부/고용노동부고객상담센터/훈령/{명}/본문.md
대통령훈령
국무총리/국무조정실/대통령훈령/{명}/본문.md

# 공공데이터 관리지침

Git 활용 방법

행정규칙 저장소는 기관 계층과 행정규칙 종류가 경로에 들어가므로 grep 명령어와 git log 명령어로 기관별 기준과 발령 이력을 좁혀 볼 수 있습니다.

행정규칙 Git 활용 
grep -rl "개인정보" 행정안전부/_본부/고시/
git log -- 고용노동부/_본부/고시/
git log -- 고용노동부/고용노동부고객상담센터/훈령/
git log --grep="^행정규칙:"

커밋 메시지

행정규칙 커밋은 발령일자를 Git author/committer date로 사용하며, 행정규칙명, 행정규칙종류, 발령기관, 제개정구분, 원문 URL을 포함합니다.

예시: 행정규칙 커밋 
행정규칙: 공공데이터 관리지침 (일부개정)

행정규칙 전문: https://www.law.go.kr/행정규칙/공공데이터관리지침

발령일자: 2026-01-01
행정규칙종류: 고시
발령기관: 행정안전부

CLI 사용법

기관명과 행정규칙 종류를 지정해 목록을 좁히고, 이름이나 경로로 전문을 조회합니다. 상세한 설치와 옵션은 cli-tools 저장소를 참고하세요.

행정규칙 CLI 
pip install legalize-cli
legalize admrules list --agency 행정안전부 --type 고시 --json
legalize admrules get "공공데이터 관리지침" --type 고시 --agency 행정안전부 --json
legalize search "개인정보" --in admrules --json

MCP 사용법

MCP에서는 admrules_list와 admrules_get 도구를 제공합니다. 특정 부처의 고시나 훈령을 찾거나, 법령·자치법규와 함께 통합 검색할 때 사용합니다. MCP 서버 설치와 클라이언트 설정은 cli-tools 저장소를 참고하세요.
