// GitHub API 기반 조회

CLI로 바로 조회합니다.

저장소를 통째로 복제하지 않아도 법령, 판례, 행정규칙, 자치법규를 같은 명령 체계로 조회할 수 있습니다. 자동화에는 --json 출력을 사용합니다. 상세한 설치와 옵션은 cli-tools 저장소를 참고하세요.

legalize-cli
$ pip install legalize-cli
$ legalize laws get 민법 --date 2015-06-01
$ legalize precedents list --court 대법원 --type 민사 --json
$ legalize ordinances list --jurisdiction 서울특별시 --type 조례 --json
$ legalize admrules list --agency 행정안전부 --type 고시 --json
$ legalize search "개인정보" --in all --json

원문 출처

공식 출처 기반 원문

국가법령정보센터 OpenAPI에서 수집한 공공 법률문서를 원문 링크와 함께 보존합니다. 법령명, 사건번호, 공포일자, 선고일자 같은 근거 필드를 원문과 함께 확인할 수 있습니다.

구조화 형식

Markdown + YAML frontmatter

날짜 기준

법령 개정, 판례 선고, 행정규칙 발령, 자치법규 공포 날짜를 Git 커밋 이력으로 남깁니다. “특정 날짜의 법령 상태” 같은 질의를 처리할 때 기준점으로 쓸 수 있습니다.

// 자주 묻는 질문

Legalize KR FAQ

처음 방문한 사용자가 Legalize KR의 범위와 활용 방식을 빠르게 확인할 수 있도록 질문 단위로 정리합니다.

Legalize KR은 무엇인가요?

Legalize KR은 대한민국 법령, 판례, 행정규칙, 자치법규를 Markdown 파일과 Git 커밋 이력으로 관리하는 오픈소스 법률 데이터셋입니다.

AI 법률 검색에 활용할 수 있나요?

네. 법률 원문과 날짜 기반 변경 이력을 함께 제공하므로 AI 법률 검색, 리걸테크 서비스, RAG 색인, MCP 도구, 법률 데이터 분석에 활용할 수 있습니다.

판례 검색과 법령 검색을 모두 지원하나요?

법령, 판례, 행정규칙, 자치법규를 각각 별도 Git 저장소로 제공하며 CLI와 MCP를 통해 같은 명령 체계로 조회할 수 있습니다.

Legalize KR이 법률 조언을 제공하나요?

아니요. Legalize KR은 공공 법률문서를 구조화해 제공하는 데이터 프로젝트이며, 개별 사건에 대한 법률 조언이나 변호사 상담을 제공하지 않습니다.
