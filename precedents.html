판례 데이터

대한민국 판례를 사건종류와 법원등급 기준으로 정리합니다. 사건번호, 선고일자, 법원명, 사건종류를 메타데이터로 보존하고 판례 원문은 Markdown으로 제공합니다.

개요

판례 데이터는 국가법령정보센터 OpenAPI의 판례 원문을 Markdown 파일로 변환한 결과입니다. 법원명과 사건번호를 기준으로 판례를 식별하고, 민사·형사·일반행정·세무·특허·가사 등 사건종류별로 분류합니다.

판례는 법령과 달리 개정되는 문서가 아니라 누적되는 문서입니다. Legalize KR은 선고일자를 기준 날짜로 사용해 Git 이력에서 언제 어떤 판례가 추가되었는지 확인할 수 있게 합니다.

데이터 구조

판례는 사건종류, 법원등급, 사건번호 순서로 배치됩니다. 같은 사건번호가 여러 법원등급에 존재할 수 있으므로 경로 전체를 안정적인 식별자로 취급합니다.

민사
민사/{법원등급}/{사건번호}.md
형사
형사/{법원등급}/{사건번호}.md
일반행정
일반행정/{법원등급}/{사건번호}.md
그 외
{사건}/{법원등급}/{사건번호}.md
판례 파일 구조 
$ tree precedent-kr/ -L 3
precedent-kr/
├── 민사/          # —개
│   ├── 대법원/     # —개
│   └── 하급심/     # —개
├── 형사/          # —개
├── 일반행정/       # —개
├── 세무/          # —개
├── 특허/          # —개
├── 가사/          # —개
└── 기타/          # —개

# 2000다10048

## 판시사항

Git 활용 방법

판례 저장소는 사건종류와 법원등급별 경로가 분리되어 있어 grep, git log, find 명령어로 원하는 범위를 좁혀 분석할 수 있습니다.

판례 Git 활용 
grep -rl "손해배상" 민사/대법원/
git log -- 민사/대법원/2000다10048.md
git log --since="2025-01-01" --until="2025-12-31"
find 민사 -name "*.md" | wc -l
git log --grep="^판례:"

커밋 메시지

판례 커밋은 선고일자를 Git author/committer date로 사용하며, 원문 URL, 법원명, 사건종류, 판례일련번호를 포함합니다.

예시: 판례 추가 커밋 
판례: 소유권이전등기등 (2000다10048)

판례 전문: https://www.law.go.kr/LSW/precInfoP.do?precSeq=81927

선고일자: 2002-09-27
법원명: 대법원
사건종류: 민사
판례일련번호: 81927

CLI 사용법

사건번호나 경로로 판례 전문을 조회하고, 법원·사건종류 기준으로 목록을 좁힐 수 있습니다. 상세한 설치와 옵션은 cli-tools 저장소를 참고하세요.

판례 CLI 
pip install legalize-cli
legalize precedents list --court 대법원 --type 민사 --json
legalize precedents get "2022다12345" --json
legalize search "손해배상" --in precedents --json

MCP 사용법

MCP에서는 precedents_list와 precedents_get 도구를 제공합니다. 통합 search 도구에서 in=precedents를 지정해 판례만 검색할 수도 있습니다. MCP 서버 설치와 클라이언트 설정은 cli-tools 저장소를 참고하세요.

활용 사례

사건종류별 분석

민사, 형사, 일반행정 등 사건종류별 판례 분포와 텍스트를 분석합니다.

법령 조문 연결

판례 본문에서 인용된 법령명을 찾아 법령 데이터와 함께 검색합니다.

판례 검색 보조

사건번호, 법원명, 키워드를 조합해 서비스의 판례 검색 색인을 만듭니다.
